So, what is THIS?

  1. In Global context: Will call the default Window object in the browser, so it won’t help ya
  2. In Object Construction: when creating a new instance of an object, this refers to the instance. This is a keyword, not a variable.
  3. In an Object Method: this refers to the object within a method. When working with multiple objects within a method, this allows you to get the correct instance.
  4. In a simple Function: I don’t use this in simple functions because it doesn’t make sense to have to take one step further to create a variable to hold the reference to this. I’d rather use an arrow function.
  5. In an Arrow Function: this keeps the context of the object (in its immediate scope).
  6. In Event Listeners: this is set to the element that fired or ‘started’ the event in the listener.

    7. TO REMEMBER:

    • You need to use bind to reference the object in an event listener
    • Bind will create a new function hat will force the this inside the function to be the parameter passed to bind(). *** bind doesn’t work with arrow functions.

Chapter 5: Objects

Creating Objects

I like how js allows objects to be created quickly without having to define a class. It’s a great way to organize all of the object’s properties. I also prefer the ES6 method of creating objects if a property key is the same as a variable name that the property value is assigned to. Example from the book:

const name = 'Iron Man';
const realName = 'Tony Stark';
// long way
const ironMan = { name: name, realName: realName };
// short ES6 way
const ironMan = { name, realName };

Accessing Properties

I prefer t use dot notation when accessing an object’s property. Working off the superman example from the book, I’d rather use superman.name than using the brackets, unless I want to use multiple properties to create a string or something:

Superman[“real” + “” + “name”]

Calling Methods

We can also call an object’s method with my preferred dot notation. Object.functionName() Objects also make it easy to check if a property or method exists. A few different ways are:

'city' in superman; << false
superman.city !== undefined; << false
superman.hasOwnProperty('city'); << false
superman.hasOwnProperty('name'); << true

Looping through, Adding, CHanging, and Deleting Properties

Using a simple for in loop will ‘loop’ or iterate through all of your object’s properties:

for(const key in superman) { console.log(key + ": " + superman[key]); }

<< "real name: Clark Kent"

console.log(\"Up, up and away!\"); }"

To add a property, simply use dot notation to add the new property to your object:

superman.city = 'Metropolis'; << 'Metropolis'

Now your superman object has a new property, city with the value ‘Metropolis’. To change a property, just change it to what you want it to be:

superman['real name'] = 'Kal-El';
<< 'Kal-El'

And a simple delete command will delete a property:

delete superman.fly << true

Built In Objects

I love built in objects like Math because it makes it so easy to use their properties without having to write your own functions. I like to use Math as an example because it’s a great visual. Let’s take a few of it’s properties as examples:

-Rounding methods:
Math.ceil will round a number UP to the next integer:
Math.ceil(4.2); << 5
Math.ceil(8); << 8
Math.ceil(-4.2); << -4
Math.floor() will round a number DOWN to the next integer:
Math.floor(4.2); << 4
Math.floor(8); << 8
Math.floor(-4.2); << -5
Math.round() method will round a number to the NEAREST integer:
Math.round(4.5); << 5
Math.round(4.499); << 4
Math.round(-4.2); << -4
And Math.trunc() will truncate the number at the decimal point:
Math.trunc(4.9); << 4
Math.trunc(-4.2); << -4

Chapter 6: Document Object Model

DOM represents an HTML document as a network of connected noes. The Html tag is the root node, and every other part of the document is a child node.

WHY?WHY? DOM allows us to access elements from an HTML doc to use it in our javascript code. For example we can create a variable called ‘body’ and assign it the value of everything inside the html body tag like this:

Const body = document.body;

TO REMEMBER:

DOM Shortcut Methods:

  • • Document.body returns the body element of a web page, as we saw in the previous example.
  • • Document.images returns a node list of all the images contained in the document.
  • • Document.links returns a node list of all the elements and elements that have an href attribute.
  • • Document.anchors returns a node list of all the elements that have a name attribute.
  • • Document.forms returns a node list of all the forms in the document.

Getting Elements

DOM makes it easy to get elements. You can get elements by their:

  • -ID document.getElementByID(‘title’);
  • -Tag Name document.getElementsByTagName(‘li’);
  • -Class Name document.getelementsByClassName(‘hero’);
  • -Using document.querySelector() allows us to use CSS to find the FIRST element in the document that matches a selector proved as an argument.
    Example, this will select the first element that has the id ‘bats’:
    Document.querySelector(‘#bats’);

Adding, Deleting and Replacing Elements

  • -Heroes.appendChild(wonderWoman);
    will append wonderWoman at the end of an unordered list.
    If you want the new element to be in between two existing elements, use insertBefore();
  • -heroes.removechild(aquaman) will remove the aquaman element from the list
  • -h1.replaceChild(newText, oldText); will replace the object ‘h1’ created with newText
