Master Mobile UX - Notes

-‘Rewire’ is the right word! I have always, always ignored instructions when told to design mobile first because I never wanted to change the way my brain worked. My brain immediately saw the desktop design, not the mobile. But then it was so difficult to change the design to fit the mobile view! I paid better attention to how I view sites on a daily basis-and the only time I used my desktop was for my classwork. For everything else I used my phone! So I’m committed to rewire-ing my brain for Mobile First! -“Just because your website is responsive, doesn’t mean it’s mobile first”…… whaaaaaat?

Remember:

  • Start small and scale up
  • Ensures that you’re really concentrating on your content and your functionality to make sure things are going to work well on small devices.

Better Mobile Design

  1. Be there
  2. Be Useful
  3. Be Quick

Reachability Matters

  • Users will use their phones one handed
  • Larger screen (tablet) make it easy to reach (thumbs are on the bottom)

Speed/Network Matters

  • Not everybody has the latest and greatest
  • 1/2 second delay = 20% drop in traffic
  • Reduce image usage
  • SVG/Webfonts are lower bandwidth
  • Optimize and Minify CSS and JS
  • GZIP files to eliminate unnecessary data

    • Perception Matters - Reduce cognitive load

    Hamburger icon vs MENU button (+20% clicks

    Mobile Friendly Forms

    • Amount of information asked of users
    • Ditch labels-have them appear in fields
    • Float labels
    • Increase field size (thumb size)
    • Autocomplete
    • Show passwords (but allow users to hide if wanted)
    • Appropriate keyboards

    Mobile Ads

    • Keep them small, simple/sleek
    • Stop using Pop-ups/overs
    • use advertising sparingly
    • Make sure it’s unobtrusive
